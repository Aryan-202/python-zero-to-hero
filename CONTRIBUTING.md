# Contributing Guidelines

We love your input! We want to make contributing to this Python course as easy and transparent as possible.

## How to Contribute

### Reporting Bugs
1. Check if the bug has already been reported
2. Create a new issue with a clear description
3. Include code examples and error messages

### Suggesting Enhancements
1. Open an issue describing your enhancement
2. Explain why this enhancement would be useful
3. Provide examples if possible

### Adding New Content
1. Fork the repository
2. Create a feature branch
3. Add your content following the existing structure
4. Test your code
5. Submit a pull request

## Code Style
- Follow PEP 8 guidelines
- Use descriptive variable names
- Include comments for complex logic
- Write docstrings for functions and classes

## Pull Request Process
1. Update documentation if needed
2. Add tests for new functionality
3. Ensure all tests pass
4. Request review from maintainers
